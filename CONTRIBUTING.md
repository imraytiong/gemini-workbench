# Contributing to Gemini Workbench

We welcome contributions to the Gemini Workbench! Whether it is bug fixes, new features, or documentation improvements, we appreciate your help.

## How to Contribute

1. **Fork the Repository**: Create a fork of the `gemini-workbench` repo.
2. **Setup**: Run `bash setup-workbench.sh` to initialize your local environment.
3. **Branching**: Use descriptive feature branches (e.g., `feat/my-new-feature` or `fix/issue-description`).
4. **Testing**: Add or update tests in `tests/` before submitting a PR.
5. **Submit a PR**: Open a pull request against the `main` branch.

## Standards
- Follow the existing project structure.
- Adhere to the defined Conductor tracks for new features.
- Ensure all tests pass (`pytest tests/`).
